16-Oct-2013 0.5.1

rexsl:run

Full name:

com.rexsl:rexsl-maven-plugin:0.5.1:run

Description:

Run WAR as a web server to enable interactive testing of a web application.

Run mvn clean package rexsl:run -Drexsl.port=9099 and open your browser at http://localhost:9099/. You will see the front page of your application. What is especially interesting is that you can make changes to static resources (xsl, css and other files in src/main/webapp) and then just refresh the page in a browser. New versions of the files will be delivered. That's how you can interactively test your XSL stylesheets.

Attributes:

  • Requires a Maven project to be executed.
  • The goal is thread-safe and supports parallel builds.

Required Parameters

Name Type Since Description
skip boolean - Shall we skip execution?
Default value is: false.
User property is: rexsl.skip.
webappDirectory String - Webapp directory.
Default value is: ${project.build.directory}/${project.build.finalName}.
User property is: rexsl.webappDirectory.

Optional Parameters

Name Type Since Description
port Integer - TPC port to bind to (by default a random port is used).
User property is: rexsl.port.
systemPropertyVariables Map 0.3 System variables to be set before running tests.

You define them in a similar way as in maven-surefire-plugin, for example you want to reconfigure LOG4J just for the tests:

<plugin>
  <groupId>com.rexsl</groupId>
  <artifactId>rexsl-maven-plugin</artifactId>
  <configuration>
    <systemPropertyVariables>
      <log4j.configuration>./x.props</log4j.configuration>
    </systemPropertyVariables>
  </configuration>
</plugin>
